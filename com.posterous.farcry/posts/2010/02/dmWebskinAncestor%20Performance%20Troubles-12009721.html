dmWebskinAncestor Performance Troubles

February 19 2010, 9:28 PM  by Geoff Bowers
dmWebskinAncestor is a system content type used to track and manage the relationship of caches in the view. Specifically it tracks where one view is embedded inside another view. This is critical for FarCry's dynamic caching model so that it knows what other views it needs to flush if a content item changes.

If your FarCry project is not performing well it may be due to the size of the dmWebskinAncestor table and/or the absence of any indices (the FarCry installer does *not* apply indices by default). To make matters worse, FarCry 5 was not as efficient as more recent versions of FarCry, tracking the embedding of views even where the view was not cached. This lead to a much bigger table and less efficient queries.

One last thing, when you are updating a project from FarCry 5.x -> 6.x you should truncate this table and apply the above indexes first.
