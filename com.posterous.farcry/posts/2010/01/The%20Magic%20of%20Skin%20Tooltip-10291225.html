The Magic of Skin Tooltip

January 20 2010, 8:37 PM  by Rob

In FarCry 6, the old extjs:toolTip tag ( http://docs.farcrycore.org/trunk/tags/farcry-core-tags-extjs-toolTip.html ) has been deprecated and replaced with the new skin:tooltip ( http://docs.farcrycore.org/trunk/tags/farcry-core-tags-skin-tooltip.html ) tag. The end result of the tags is similar, but how you go about using them is a bit different.

With the old extjs:toolTip tag, you set the attribute "toolTip" to the tool tip's text, and then add the content the tool tip applies to within the body of the extjs:toolTip tag. skin:tooltip works differently.

With skin:tooltip, you create a DOM element of some kind, and give that element an id attribute. You can set the "message" attribute, or use the tags content area, to set the tool tip text. You then link the tool tip to the DOM element by setting the "selector" attribute to match the id of the DOM element you created (See example A and B below).

If you are astute, you caught that the "selector" attribute begins with a hash. The reason the hash is necessary is the underpinnings of the new skin:tooltip is jQuery. The "selector" attribute gets passed through to the tool tip jQuery code when rendering the page. In jQuery using a hash indicates an id attribute of a DOM element.

Since you can pass any jQuery selector into the skin:tooltip tag, you can do interesting things with the tool tips. For example, you can set tool tips on anything with a specific class (See example C).
