Multi-Site Options

January 8 2010, 12:32 AM  by Geoff Bowers
FarCry has two distinct approaches to managing multiple web sites under a single installation: “multiple sites” and “sub-sites”.

As a general rule of thumb, if sites do not share content then they should be treated as distinct projects - the multiple sites approach. On the other hand, if a site shares a significant amount of content with another site then you may consider creating a single application with embedded sub-sites.

--// Multiple Sites

The “Multiple Site” approach follows creating distinct projects for each web application. These sites may share a common code base through the use of plugins.  Specifically each application has its own logical database.

--// Sub-Sites

The “Sub-Site” approach uses a single application to manage all the different sub-sites. All content is housed in a single logical database. The sub-sites each load a unique set of plugins at run time; the combination of plugins determining the look and feel and specific functionality available to each sub-site.

The information hierarchy of each sub-site is managed in a separate branch of the site overview tree, allowing each subsite to have its own unique sitemap. Dynamic content is typically segmented between sub-sites using categories.

Daemon has a "Sub Sites" plugin available for commercial license holders that greatly simplifies the configuration and development of sub-site installations. Contact us if you're interested.
